Contrastive Learning (6) — Contrastive Learning with positive/negative sampling

In addition to the lack of high-level semantic representations, the sampling of positive and negative samples for instance discrimination tasks also has obvious problems. Positive samples are all from its own data augmentation, while negative samples are directly sampled from a large memory (within the same batch, memory bank or a momentum encoder). The problem with this is that negative samples may have positive sample, which they are the same class, but are different instances. Another problem is that hard negative samples and easy negative samples have the same weights in InfoNCE loss , which make model too simple if it focus on too many easy negative samples. In this chapter, we will see many interesting works with different methods on solving sampling problem in CL.

What Should Not Be Contrastive In Contrastive Learning

Link: https://arxiv.org/abs/2008.05659

Motivation:

This paper proposed that data augmentation may hurt image representations, especially when they are used on downstream tasks. For example, a bird identification task is based on representation vectors by color augmentation, or a animal identifacation task is based on representation vectors by rotation augmentation.

Sounds reasonable, and it did!

Result:

Table 1 showes that add rotation augmentation on MoCo will significantly reduces its accuracy on IN-100(ImageNet-100) this dataset. And their model LooC has a better result.

Method:

And add all these \(Z\) regions into loss function:

\[\mathcal{L}_{q}=-\frac{1}{n+1}\left(\log \frac{E_{0,0}^{+}}{E_{0,0}^{+}+\sum_{k^{-}} E_{0,0}^{-}}+\sum_{i=1}^{n} \log \frac{E_{i, i}^{+}}{\sum_{j=0}^{n} E_{i, j}^{+}+\sum_{k^{-}} E_{i, i}^{-}}\right)\]

where \(E_{i, j}^{\{+,-\}}=\exp \left(\boldsymbol{z}_{i}^{q} \cdot \boldsymbol{z}_{i}^{k_{j}^{\{+,-\}}} / \tau\right)\)

Insights:

I think the movitation is quite reasonable, but I cant agree with the solving method which aggregates all different augmentation infomations into the same loss function. It's more like an improvement in robustness, rather than aiming at adapting with specific downstream tasks.

By the way, I dont like the section of method description. It is a very simple flow but uses so many mathematic symbols, while the most important part of training is simple and hard to understand. This paper also does not provide open source code for me to understanding.
