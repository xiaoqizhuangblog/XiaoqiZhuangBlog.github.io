Joins vs Sub-queries

Advantages of join queries – The join implementation can display data from all tables in the FROM clause whereas, the sub-query implementation can only display data from the table(s) in the outer query.

Advantages of nested sub-queries – The ability to calculate an aggregate value on the fly and feed it back to the outer query for comparison is an advantage sub-queries have over joins.

Conclusion:

– Use joins when you are displaying results from multiple tables.

– Use sub-queries when you need to compare aggregates to other values.
