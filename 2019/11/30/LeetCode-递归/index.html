递归

递归是一种解决问题的有效方法。在递归过程中，函数将自身作为子例程调用

此博客记录学习LeetCode新手教程“探索递归”的总结。主要讲述了递归的原理，递归复杂度计算，记忆化技术。包含8道关于递归的练习题及解（杨辉三角，幂计算，爬楼梯等）。

第一题：相反字符串

题目：以相反的顺序打印字符串

例：

思路

将字符串顺序相反用递归的方式也可以理解为：

  1. 将首尾对调
  2. 把首尾去除后得到新的首尾（原来的第二位和倒数第二位），对调新的首尾
  3. 继续步骤2
  4. 直到没有字母或者只剩下最后一个字母时退出（退出条件）

代码

代码解释：

第6行：创建一个辅助函数进行位置的对调

​ start是首，end是尾，ls即s

第7行：递归的终止条件

第11行：对调的过程

第13行：递归的过程

第15行：把输入字符串 ’s’ 放入我们的辅助函数helper中进行工作

演示步骤

将 ["h","e","l","l","o”] 输入后的步骤：

  1. 到第15行 helper(0, len(s)-1, s), 即初始 start = 0, end = len(s)-1 = 4, ls = s = ["h","e","l","l","o”]
  2. 验证此时 start (0) < end(4), 进行换位（第13行）
  3. 此时 ls 经过第11行变为 ["o","e","l","l”,"h”]
  4. 第13行进行递归，继续执行 helper函数，此时 start = 1， end = end - 1 = 3, ls = ["o","e","l","l”,"h”]
  5. 验证递归结束条件 start(1) < end(3), 没有到退出条件，进行换位
  6. 此时 ls 变为 ["o”,"l","l”,"e”,"h”]
  7. 继续第13行的递归，此时start = 2，end = 2，ls = ["o”,"l","l”,"e”,"h”]
  8. 验证此时 start(2) = end(2)，结束递归，return 此时 ls = ["o”,"l","l”,"e”,"h”]
  9. 第十六行，return s（s 就是 ls 因为在第一次helper中，s 是参数）

最后结果为 ["o”,"l","l”,"e”,"h”] ，是我们想要的结果。

例：

思路

杨辉三角有以下这种公式：

$ f(i, j)=f(i-1, j-1)+f(i-1, j)$

\(f(i, j)=1 \quad \text { where } \quad j=1 \quad \text { or } j=i\)

根据第二个条件，我们可以看出杨辉三角中的两边都是1。

所以第一步我们可以创建一个全部为1的三角形。

再根据第一个条件，为三角形内部填充值。

代码

第8行代码构造了一个全部为1的三角形。

result = [[1]*(i+1) for i in range(numRows)]

Tips: 一个list乘以一个int值，可以变为一个新list，数量为之前list的int倍。

第四题：斐波那契数列

斐波那契数列：\(F(n)=F(n-1)+F(n-2)\), 其中 \(F(0)=0, F(1)=1\)

初步代码

（Memorization）记忆化技术

计算f(4)，我们发现使用了2遍 f(2)，f(2)需要进行 f(0)+f(1)的计算操作。

可想而知，当我们计算 f(100) 时候，这种重复的操作会非常的巨大。

不信可以试试用上述代码计算 fibonacci(100), 我的电脑算不出来。。。

此时我们有一种技术称为 记忆化，其核心思想为将出现过一次的数据记录在一个hash表中，每当再次出现需要这个数据，直接调用而不需要再进行重复计算。

优点：速度会比原来快很多，我的电脑立马就算出了fibonacci(100)。

缺点：会占用内存，当你的递归函数是数值很多，但是重复操作很少时，反而弄巧成拙。

优化代码

第6行cache为一个字典

第17行将递归过程中出现的值放进此字典中

第8行，如果此值在字典中则直接调用，避免重复计算

第五题：爬楼梯

题目：假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

​ 每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

例：

思路

我们爬 n 阶楼梯的方法可以有爬（n-1）阶和（n-2）阶楼梯的方法中得来。

例如我们爬 3 阶楼梯，如果我们已知爬 2阶楼梯的方法有2种，爬1阶楼梯的方法有1种。那么我们爬 3 阶楼梯的方法就有3种（2+1）。

因为我们只需要在已知 2阶楼梯的方法下 走一步，或者在已知 1阶楼梯的方法下 走2步。

注意这里只是步数增加了，然而方法的次数还是一样的

例如走 2阶楼梯的方法为 ： 1+1 和 2，

那么根据走2阶楼梯方法转换来的三阶楼梯方法为：1+1+1 和 2+1。

还是2种方法。

所以我们得出推论： num(n) = num(n-1) + num(n-2)

终止条件为：num(2) = 2 and num(1) = 1

但是很多情况下，一个递归函数可能一次同时调用2次递归函数，并且分布不均匀（描述可能有歧义，但是分析斐波那契额数列的情况便可知）。

例如我们想要计算 fibonacci(4)的\(R\), 并没有一个统一的公式。此时我们需要执行树帮助判断。

此为 \(f(4)\)的执行树。

我们可以发现，递归关系的执行树为n叉树。n为调用递归函数的次数。而n叉树的节点数即为\(R\)。

此例子中\(n=2\), 因为递归关系为return fibonacci(n-1) + fibonacci(n-2)

所以我们可知完全二叉树的节点数为 \(2^n -1\) （很多时候不是完全二叉树，但是复杂度看上限）。

$ R = 2^n$

\({\mathcal{O}(s)} = {\mathcal{O}(1)}\)

即\({\mathcal{O}(T)} = R * {\mathcal{O}(s)} = {\mathcal{O}(2^n)}\)

可以看出这个复杂度非常的高，所以需要用记忆化。

记忆化

现在我们来算算计算经过记忆化后的斐波那契数列的时间复杂度。

我们知道计算一个斐波那契数\(f(n)\)，可以用前面\(n-1\)个数得出来。

比如计算\(f(4)\), 我们知道前面的3个数：1，1，2。 就可以算出来 \(f(4) = 3\)

因为记忆化已经帮助了我们记录前 n 个数。

因此 \({\mathcal{O}(T)} = R * {\mathcal{O}(s)} = n * {\mathcal{O}(1)} = {\mathcal{O}(n)}\)

第六题：二叉树的最大深度

给定一个二叉树，找出其最大深度。

二叉树的深度为根节点到最远叶子节点的最长路径上的节点数。

说明: 叶子节点是指没有子节点的节点。

例：

给定二叉树 [3,9,20,null,null,15,7]，如下图：

返回它的最大深度：3

思路

例子中的最大深度为 3-20-15 或者3-20-7 这两条线。以此可见找到最大深度其实是一个递归的过程。

一些基本知识：

叶子节点的深度为1。 None的深度为0。

父节点的深度为其最大的子节点的深度+1。（因为子节点有2个，深度可以不一样，最大深度取其中较大的子节点深度再加1）

步骤：

  1. 找 3的最大深度，我们计算它的子节点 9和20的最大深度。

  2. 计算9的最大深度，由于9是叶子节点（没有子节点），9的深度为 1。

  3. 计算20的最大深度，计算它的子节点15和7的最大深度。

  4. 15和7都是叶子节点，其深度为1。

  5. 因此20的深度为2。

  6. 9的深度为1，20的深度为2，因此 3的深度为 3

    3是由于\(max(depth(20),depth(9)) + 1 = 2+1 = 3\)。不是因为\(depth(20)+depth(9) = 3\)

递归终止条件：当节点为none节点时，返回深度为0

递归调用条件：当节点是非空节点时，返回深度为\(max(depth(root.left),depth(root.right))+1\)
